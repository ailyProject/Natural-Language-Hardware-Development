# 第 7 章：传感器数据读取和处理

## 本章要解决什么问题

传感器能把真实世界变成数据。温度、湿度、光照、距离、声音、姿态，这些都可以通过传感器读出来。

但读到数据只是第一步。真实传感器会有波动，会有偶发错误，也可能需要校准。本章要学习的是：怎样让 AI 帮你读传感器，同时学会检查、处理和验证数据。

## 7.1 传感器项目的基本流程

一个传感器项目通常包含五步：

1. 安装或选择对应库。
2. 正确接线。
3. 初始化传感器。
4. 定时读取数据。
5. 输出、判断或控制其他模块。

用自然语言写需求时，也要按这个顺序说清楚。

## 7.2 数据为什么会不稳定

很多新手会以为传感器读数应该一直很准。实际上，真实数据经常会跳动。

常见原因包括：

- 传感器本身有误差。
- 供电不稳定。
- 线太长或接触不良。
- 环境本来就在变化。
- 采样太快，数据还没稳定。
- 模块库返回了异常值。

所以，学习传感器不要只停在“读出一个数字”，还要学会看数据、判断数据、处理数据。

## 7.3 用自然语言描述传感器任务

描述传感器项目时，建议包含这些信息：

1. 传感器型号。
2. 使用接口类型，如 I2C、模拟量、数字引脚。
3. 接线方式。
4. 多久读取一次。
5. 串口或屏幕输出什么。
6. 是否需要阈值判断。
7. 是否需要滤波或异常处理。

示例：

```text
我使用 Arduino Uno 和 DHT20 温湿度传感器。DHT20 使用 I2C 接口，SDA 接 A4，SCL 接 A5，VCC 接 5V，GND 接 GND。请每 2 秒读取一次温度和湿度，在串口输出。如果读取失败，请输出“SENSOR ERROR”。温度超过 30 度时点亮 LED。
```

## 7.4 实验项目一：温湿度监测

### 提示词

```text
我使用 Arduino Uno 做温湿度监测项目。

硬件连接：
1. 使用 DHT20 温湿度传感器。
2. DHT20 使用 I2C 接口。
3. SDA 接 A4，SCL 接 A5，VCC 接 5V，GND 接 GND。
4. LED 使用板载 LED。

程序行为：
1. 启动串口，波特率 9600。
2. 每 2 秒读取一次温度和湿度。
3. 串口输出温度和湿度。
4. 温度超过 30 度时点亮 LED，否则熄灭 LED。
5. 如果读取失败，串口输出“SENSOR ERROR”。

请生成 Blockly 程序逻辑，说明需要安装的库，解释 Arduino/C++ 代码思路，并给出接线检查清单。
```

> 图片占位 7-2：DHT20 与 Arduino Uno 的 I2C 接线图。

### 学习重点

这个实验不是为了记住某个传感器库的全部函数，而是为了理解：

- 传感器通常要初始化。
- 读取数据需要间隔，不要无限快速读取。
- 读数要通过串口观察。
- 读数可以触发输出动作。
- 读取失败也要有提示。

## 7.5 实验项目二：光照阈值提醒

模拟光敏传感器适合练习阈值判断。

提示词示例：

```text
我使用 Arduino Uno 和模拟光敏传感器模块。光敏传感器 AO 接 A0，VCC 接 5V，GND 接 GND。LED 接 D9。请每 200 毫秒读取一次光照值，并在串口输出。请设置一个阈值变量，当光照值低于阈值时点亮 LED。请提醒我如何根据串口数据调整阈值。
```

这类项目适合先自己观察串口数据，再决定阈值。例如亮环境读数大约 800，遮住传感器读数大约 200，就可以把阈值先设为 500。

## 7.6 简单滤波：让数据别跳得太厉害

如果传感器数值跳动明显，可以用平均值让它平滑一点。比如连续读 5 次，求平均。

自然语言可以这样说：

```text
请给当前光敏传感器项目增加简单平均滤波。每次连续读取 5 个模拟值，计算平均值，再用平均值判断是否低于阈值。串口同时输出原始值和平均值。
```

> 图片占位 7-3：原始数据曲线与平均滤波后曲线对比图。

滤波不是越多越好。次数越多，数据越平稳，但反应越慢。你可以观察 3 次平均、5 次平均、10 次平均的差别。

## 7.7 异常值和校准

有些传感器偶尔会读到明显不合理的值。例如距离传感器突然返回 0，温度突然变成 -100。程序不能完全相信每一个数。

可以要求 AI 增加异常处理：

```text
请在当前传感器读取程序中加入异常值判断。如果温度小于 -20 或大于 80，就认为读数异常，不更新显示，只在串口输出“INVALID DATA”。
```

校准则是把传感器读数调整得更接近真实值。例如温度计显示 26.0 度，而传感器长期显示 27.2 度，可以加一个偏移量 `-1.2`。

## 7.8 传感器项目检查清单

- 传感器型号是否说清楚。
- 接口类型是否确认，如 I2C、模拟量或数字引脚。
- VCC 电压是否匹配。
- GND 是否连接。
- I2C 的 SDA、SCL 是否接对。
- 是否安装了对应库。
- 是否设置了合理采样间隔。
- 串口是否输出原始数据。
- 是否处理读取失败或异常值。

## 7.9 常见问题

### 一直读不到传感器

检查供电、GND、接口接线、库是否安装、I2C 地址是否正确。

### 数据一直是 0 或固定值

可能是引脚接错、读取函数用错、传感器没有供电，或模块输出端口接错。

### 数据跳动大

检查供电和接线，适当增加平均滤波，不要采样太快。

### AI 生成的库不可用

把编译错误和库名称发给 AI，让它基于当前项目修正，不要直接让它重写整个项目。

## 7.10 本章练习

1. 用光敏传感器做一个自动小夜灯。
2. 给温湿度项目增加“温度过高报警”。
3. 让 AI 为传感器项目增加 5 次平均滤波。
4. 记录一组串口数据，自己判断合适阈值。
5. 写一段完整问题描述，让 AI 排查“传感器读数一直不变”。

## 7.11 本章小结

传感器让硬件项目能感知环境。自然语言可以帮你快速生成读取程序，但真正重要的是读懂数据：它是否稳定、是否合理、是否需要滤波、是否能触发正确动作。

以后做任何传感器项目，都要养成先串口输出原始值的习惯。有了数据证据，AI 和人都更容易判断问题。
