# 第 8 章：执行器控制和电源安全

## 本章要解决什么问题

传感器负责“感知”，执行器负责“动作”。舵机会转，电机会动，继电器会吸合，灯带会亮，蜂鸣器会响。

执行器让项目更有趣，也更容易出问题。因为很多执行器需要的电流比开发板引脚能提供的大得多。本章重点不是让你害怕，而是让你知道：让东西动起来之前，先确认电源和安全。

## 8.1 执行器和 LED 有什么不同

LED 消耗的电流很小，而且通常只需要一个限流电阻。执行器就复杂一些：

- 舵机启动时可能需要较大电流。
- 电机容易产生干扰和反向电动势。
- 继电器可能控制外部电路。
- 灯带电流可能远超开发板能力。

所以，执行器项目里最重要的问题不是“代码怎么写”，而是“能不能这样供电”。

## 8.2 先记住三条安全规则

1. 不要用开发板 IO 引脚直接给大功率执行器供电。
2. 使用外接电源时，控制信号和开发板通常需要共地。
3. 入门实验不要直接控制市电负载。

如果不确定，就把模块型号、电源规格和接线描述发给 AI，让它先列风险，再生成程序。

## 8.3 用自然语言描述执行器任务

执行器提示词要说清楚：

1. 执行器是什么。
2. 信号线接哪个引脚。
3. 电源怎么接。
4. 什么时候动作。
5. 动作持续多久或到什么位置。
6. 是否需要停止条件和保护逻辑。

示例：

```text
我使用 Arduino Uno 和一个 SG90 舵机。舵机信号线接 D9，VCC 使用外接 5V 电源，GND 与 Arduino GND 共地。请让舵机在 0 度、90 度、180 度之间依次转动，每个位置停 1 秒。请提醒我电源接线注意事项。
```

## 8.4 实验项目一：舵机角度控制

### 接线说明

常见 SG90 舵机线色可能不同，通常是：

```text
棕色或黑色 -> GND
红色 -> 5V
橙色或黄色 -> 信号引脚 D9
```

如果使用外接 5V 电源，外接电源 GND 要和开发板 GND 连接在一起。

> 图片占位 8-2：舵机使用外接 5V 电源并与开发板共地的接线图。

### 提示词

```text
我使用 Arduino Uno 和 SG90 舵机做角度控制实验。

硬件连接：
1. 舵机信号线接 D9。
2. 舵机 VCC 使用外接 5V 电源。
3. 舵机 GND、外接电源 GND 和 Arduino GND 需要共地。

程序行为：
1. 舵机依次转到 0 度、90 度、180 度。
2. 每个角度停 1 秒。
3. 串口输出当前目标角度。

请生成 Blockly 程序逻辑，说明需要的库，解释 Arduino/C++ 代码思路，并列出电源安全检查清单。
```

### 代码思路

```cpp
#include <Servo.h>

Servo servo;

void setup() {
  Serial.begin(9600);
  servo.attach(9);
}

void loop() {
  servo.write(0);
  Serial.println("angle=0");
  delay(1000);

  servo.write(90);
  Serial.println("angle=90");
  delay(1000);

  servo.write(180);
  Serial.println("angle=180");
  delay(1000);
}
```

## 8.5 实验项目二：小风扇自动控制

小风扇或直流电机通常不能直接接开发板引脚。入门时可以使用带驱动的模块，或者先使用经过确认的安全接线方案。

提示词示例：

```text
我想用 Arduino Uno 控制一个 5V 小风扇。请先说明为什么不能把风扇直接接到 Arduino IO 引脚。假设我使用一个低压电机驱动模块，控制引脚接 D6，外接 5V 电源供电，GND 共地。请生成温度超过 30 度时打开风扇，否则关闭风扇的 Blockly 程序，并列出接线安全检查清单。
```

这个提示词先让 AI 解释风险，再让它生成程序，比直接说“控制风扇”安全得多。

## 8.6 实验项目三：继电器低压控制

继电器可以控制另一条电路，但入门练习建议只接低压负载，甚至只观察继电器模块指示灯。

提示词示例：

```text
我使用 Arduino Uno 和一个 5V 继电器模块。继电器 IN 接 D7，VCC 接 5V，GND 接 GND。请让继电器每 2 秒切换一次，并在串口输出状态。请特别说明入门实验不要连接市电负载，只观察模块指示灯或低压负载。
```

## 8.7 执行器项目的保护逻辑

执行器项目要尽量避免“一直动作”。可以加入保护条件：

- 舵机角度限制在安全范围。
- 电机运行一段时间后自动停止。
- 温度传感器异常时关闭风扇。
- 按下急停按钮后所有输出关闭。
- 串口输出当前状态和异常原因。

自然语言可以这样说：

```text
请给当前风扇控制项目增加保护逻辑。如果传感器读取失败，风扇关闭；如果风扇连续运行超过 5 分钟，串口输出提醒；按下 D2 急停按钮时立即关闭风扇。
```

## 8.8 执行器安全检查清单

- 执行器电压是否匹配。
- 执行器电流是否超过开发板能力。
- 是否需要驱动模块、三极管、MOS 管或继电器模块。
- 使用外接电源时是否共地。
- 是否有停止条件。
- 是否有异常处理。
- 继电器是否只用于低压安全实验。
- 串口是否输出动作状态。

## 8.9 常见问题

### 舵机抖动

常见原因是供电不足、GND 没有共地、信号线接触不良。

### 电机一转开发板重启

多半是电机电流太大或供电干扰，不能直接用开发板供电。

### 继电器逻辑反了

有些继电器模块是低电平触发。让 AI 根据模块说明把 HIGH/LOW 逻辑反过来。

### 执行器不动作但串口正常

说明程序可能在运行，重点检查供电、驱动模块、信号线和 GND。

## 8.10 本章练习

1. 让舵机根据电位器角度变化。
2. 让温度超过阈值时打开风扇，低于阈值时关闭。
3. 给执行器项目增加急停按钮。
4. 让 AI 根据你的接线描述生成安全检查清单。
5. 写一段问题描述，让 AI 排查“舵机抖动”的原因。

## 8.11 本章小结

执行器让硬件项目真正动起来，但它也最需要工程安全意识。自然语言可以帮你描述动作和保护逻辑，但供电、电流、共地、驱动模块这些问题必须认真检查。

记住一句话：让执行器动作之前，先确认它能不能安全地这样供电。
