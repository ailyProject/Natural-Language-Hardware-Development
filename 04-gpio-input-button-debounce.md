# 第 4 章：GPIO 输入、按键和消抖

## 本章要解决什么问题

上一章我们让开发板控制外部设备，这一章反过来，让外部设备告诉开发板发生了什么。最常见的输入就是按键。

按键看起来简单，但它会带出几个非常重要的概念：输入电平、上拉电阻、条件判断、状态变化和消抖。

> 图片占位 4-1：按键输入到开发板，再控制 LED 输出的流程图。

## 4.1 输入和输出有什么不同

输出是开发板主动给引脚高低电平，输入是开发板读取引脚当前是什么状态。

按键项目里，开发板通常会问：

```text
这个引脚现在是 HIGH 还是 LOW？
```

然后根据结果决定要不要点亮 LED、切换模式、计数或报警。

## 4.2 为什么要上拉或下拉

如果一个输入引脚什么都不接，它可能会乱跳，这叫“悬空”。悬空引脚读到 HIGH 或 LOW 都有可能，程序就会变得不稳定。

所以按键输入通常需要上拉或下拉，让引脚在没有按键动作时也有一个确定状态。

入门推荐使用内部上拉：

```text
引脚使用 INPUT_PULLUP
按键一端接引脚
按键另一端接 GND
松开时读 HIGH
按下时读 LOW
```

> 图片占位 4-2：内部上拉按键接线图，标注“松开 HIGH，按下 LOW”。

## 4.3 用自然语言描述输入行为

描述按键项目时，要说清楚：

1. 按键接在哪个引脚。
2. 使用内部上拉还是外部电阻。
3. 按下时做什么。
4. 松开时做什么。
5. 是否只在状态变化时触发。
6. 是否需要消抖。

示例：

```text
我使用 Arduino Uno。按键接 D2，使用内部上拉，按键另一端接 GND。LED 使用板载 LED。按下按键时 LED 点亮，松开时 LED 熄灭。请加入简单消抖，避免按键抖动导致串口重复输出。
```

## 4.4 实验项目一：按下亮，松开灭

### 提示词

```text
我使用 Arduino Uno 做按键控制 LED 实验。

硬件连接：
1. 按键接 D2，使用内部上拉。
2. 按键另一端接 GND。
3. LED 使用板载 LED。

程序行为：
1. 按下按键时 LED 点亮。
2. 松开按键时 LED 熄灭。
3. 串口波特率 9600。
4. 状态变化时输出按键状态和 LED 状态。

请生成 Blockly 程序逻辑、Arduino/C++ 代码思路和接线检查清单。
```

### 代码思路

```cpp
const int buttonPin = 2;
int lastButtonState = HIGH;

void setup() {
  Serial.begin(9600);
  pinMode(buttonPin, INPUT_PULLUP);
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  int buttonState = digitalRead(buttonPin);

  if (buttonState == LOW) {
    digitalWrite(LED_BUILTIN, HIGH);
  } else {
    digitalWrite(LED_BUILTIN, LOW);
  }

  if (buttonState != lastButtonState) {
    Serial.println(buttonState == LOW ? "PRESSED" : "RELEASED");
    lastButtonState = buttonState;
  }

  delay(20);
}
```

这里的 `lastButtonState` 用来记录上一次状态。只有状态变化时才输出串口，避免日志刷得太快。

## 4.5 什么是按键消抖

真实按键不是理想开关。按下的一瞬间，金属触点可能会快速抖动，开发板可能读到一串 HIGH、LOW、HIGH、LOW。

如果程序没有处理，就可能把一次按键误认为多次。

简单消抖方法：

- 读到状态变化后等待 20 到 50 毫秒。
- 再读一次确认状态。
- 只在确认变化后执行动作。

## 4.6 实验项目二：按一下切换一次 LED

这个项目比“按下亮，松开灭”更接近真实交互。每按一次，LED 状态切换一次。

提示词示例：

```text
请在当前 Arduino Uno 按键项目中，把逻辑改成“每按下一次按键，LED 状态切换一次”。按键接 D2，使用内部上拉，LED 使用板载 LED。请加入简单消抖，并且只在按下瞬间切换，不要在长按时连续切换。
```

这个提示词里最关键的是“只在按下瞬间切换”和“长按时不要连续切换”。如果不说清楚，AI 可能会生成按住一直切换的程序。

## 4.7 实验项目三：长按和短按

长按、短按是很多硬件交互的基础。例如短按切换模式，长按清零计数。

提示词示例：

```text
我想在按键项目中识别短按和长按。按键接 D2，使用内部上拉。短按时切换 LED 状态，长按超过 2 秒时让 LED 快速闪三次。请生成 Blockly 程序，并解释需要记录哪些变量。
```

这个任务会用到按下时间、松开时间和时间差，适合在已经理解基本按键后再做。

## 4.8 输入类项目的检查清单

- 输入引脚是否设置正确。
- 是否使用上拉或下拉，避免引脚悬空。
- 内部上拉时，是否知道按下通常是 LOW。
- 按键另一端是否接到 GND。
- 是否需要消抖。
- 串口是否能输出按键状态。
- 触发动作是按住触发，还是按下瞬间触发。

## 4.9 常见问题

### 按键没按也触发

多半是引脚悬空，或上拉/下拉配置不对。

### 按一次触发很多次

可能是按键抖动，或者程序写成了按住时持续触发。

### 逻辑反了

内部上拉时，按下通常是 LOW。可以让 AI 按这个规则检查条件判断。

## 4.10 本章练习

1. 按键按下时蜂鸣器响，松开时停止。
2. 按一下切换 LED 状态，并在串口输出切换次数。
3. 双击按键让 LED 快闪三次。
4. 让 AI 解释内部上拉和外部下拉的区别。

## 4.11 本章小结

输入让硬件项目有了交互。按键虽小，却能帮助你理解输入电平、条件判断、状态变化和消抖。用自然语言描述输入项目时，一定要说清楚“接在哪里、按下是什么状态、什么时候触发、是否消抖”。
