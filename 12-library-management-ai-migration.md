# 第 12 章：库管理和 AI 转库

## 本章要解决什么问题

嵌入式开发里，很多功能不是从零开始写的。传感器读取、屏幕显示、网络通信、舵机控制，通常都有现成库。

库能节省大量时间，但也会带来新问题：装哪个库、版本是否匹配、示例代码怎么看、函数怎么用、Blockly 里有没有对应积木。

本章学习如何用 Aily Blockly 管理库，并借助 AI 读懂库文档、示例代码，甚至把已有 Arduino 库转换成更适合 Blockly 使用的能力。

和普通问答不同，安装库、生成库、转换库通常需要 AI 读取或写入项目文件，因此建议切换到代理模式再操作。代理模式会消耗更多 token，但它适合项目分析、自动编程、错误修复和转换库这类需要动项目的任务。

## 12.1 什么是库

库可以理解成别人提前写好的功能包。比如温湿度传感器库可能已经帮你处理了通信细节，你只需要调用“读取温度”和“读取湿度”。

入门时，先理解三件事就够了：

- 库不是魔法，它只是提前写好的代码。
- 使用库前要安装，并确认项目能找到它。
- 库通常有 README、示例代码和 API 说明。

## 12.2 为什么库管理很重要

传统 Arduino IDE 里，库常常是全局安装的。不同项目可能共用同一个库版本，项目多了之后容易混乱。

Aily Blockly 采用更工程化的项目管理思路，尽量让开发板、库和项目绑定起来。这样一个项目能编译通过，换到另一个项目时不容易互相影响。

你可以把这件事理解成一句话：

```text
项目需要什么库，就在项目里说清楚，不要靠电脑里碰巧装过什么。
```

## 12.3 用自然语言分析库

当你拿到一个新库，不要急着拖积木或复制代码。可以先让 AI 帮你读文档：

```text
请帮我分析这个 Arduino 库的 README 和示例代码。请告诉我：
1. 这个库用来控制什么硬件。
2. 需要哪些接线。
3. 初始化函数是什么。
4. 常用读取或控制函数有哪些。
5. 最小可运行示例是什么。
6. 如果要做成 Blockly 积木，应该提供哪些积木。
```

这个提示词适合你在使用新模块前快速了解库。

## 12.4 实验项目一：安装并使用一个传感器库

### 提示词

```text
我想在当前项目中使用一个温湿度传感器库。请先说明我应该检查哪些内容：
1. 库是否已经安装。
2. 库版本是否适合当前开发板。
3. 示例代码如何运行。
4. 编译失败时应该看哪些报错。

然后请基于最小示例，生成一个每 2 秒读取温湿度并输出到串口的 Blockly 程序逻辑。
```

### 学习重点

你需要知道使用库不是“点安装就结束”。真正的流程是：

1. 找到合适库。
2. 看示例。
3. 跑最小示例。
4. 再改成自己的需求。

## 12.5 示例代码怎么读

示例代码通常有三类信息：

- 引入库：例如 `#include <xxx.h>`。
- 初始化：通常在 `setup()` 里。
- 使用函数：通常在 `loop()` 里读取数据或控制模块。

可以让 AI 逐行解释：

```text
请用新手能理解的方式解释这个 Arduino 示例代码。请按“引入库、创建对象、初始化、循环读取、串口输出”几个部分说明，不要只逐行翻译。
```

## 12.6 什么是 AI 转库

有些 Arduino 库没有现成 Blockly 积木。AI 转库的思路是：把器件型号、库文档、示例代码、网页、GitHub 链接或本地源码路径提供给 AI，让 AI 帮忙分析应该生成哪些积木，并尝试生成适合 Aily Blockly 使用的专用库。

你可以提供下面任意一种信息：

- 器件型号，例如 `L298N`。
- Markdown 格式文档或文档网页。
- GitHub 仓库链接。
- 本地源码路径，推荐使用这种方式，可以是 Arduino 库，也可以是其他框架或其他芯片平台的源码。

本地源码转换时，可以这样写：

```text
"C:\Users\coloz\Downloads\xxx-lib" 是 xxx 库源码，将这个库转换为 Aily Blockly 库。
```

等待转换完成后，AI 会提示你进行安装。安装前仍然要检查库名、目标硬件、示例程序和生成的积木是否符合当前项目需求。

例如一个温湿度库，可能需要这些积木：

- 初始化温湿度传感器。
- 读取温度。
- 读取湿度。
- 判断读取是否成功。
- 设置采样间隔。

这样以后就不需要直接记函数名，而是通过更直观的积木使用库能力。

如果任务比较复杂，或者当前没有现成库可以满足需求，可以在提示词里补充一句：

```text
如果现有库无法满足该需求，可考虑先生成一个支持库，然后再使用该库进行编程。
```

这会提醒 AI 先把任务拆成“生成支持库”和“使用支持库编程”两个阶段，适合底层模块、复杂外设或长程任务。

## 12.7 设计 Blockly 积木时要注意什么

好的积木应该像自然语言一样清楚：

- 积木名称要说明做什么。
- 参数不要太多，常用配置优先。
- 返回值要容易理解。
- 错误状态要能处理。
- 示例要能直接运行。

不要把 C++ 函数原样搬成一堆难懂积木。Blockly 的价值是降低理解门槛。

## 12.8 库相关常见问题

### 编译提示找不到库

检查库是否安装到当前项目，库名是否正确，示例代码中的 `#include` 是否匹配。

### 函数不存在

可能是库版本不同，示例来自旧版本或新版本。把报错和库版本发给 AI 分析。

### 库和开发板不兼容

有些库只支持特定芯片或架构。安装前要看说明。

### 示例能跑，自己项目不能跑

先对比初始化顺序、引脚、对象名称和库函数调用位置。

## 12.9 本章练习

1. 安装一个传感器库并运行最小示例。
2. 让 AI 解释一个库的 README。
3. 从示例代码中找出初始化函数和读取函数。
4. 设计 3 个适合 Blockly 的积木名称。
5. 把编译错误发给 AI，让它判断是否是库版本问题。

## 12.10 本章小结

库是嵌入式开发的重要复用方式。自然语言和 AI 能帮助你读懂库、使用库、迁移库，但前提是项目依赖、库版本和示例代码要清楚。

学会用库，你就能从简单 GPIO 项目走向更丰富的真实硬件应用。
