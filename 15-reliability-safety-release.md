# 第 15 章：可靠性、安全和发布前检查

## 本章要解决什么问题

一个项目“能跑”不等于“可靠”。展示时能亮一次灯，不代表它可以长时间运行；桌面上能转一次舵机，不代表装进作品后仍然安全。

本章把项目从“做出来”推进到“检查过”。你需要学会在发布或展示前做安全检查、功能测试、异常测试和说明文档。

## 15.1 AI 可以列清单，但不能替你确认

AI 很适合帮你生成检查清单，比如：

```text
请根据我的环境监测站项目，生成发布前检查清单，覆盖接线、电源、传感器异常、显示、串口日志和用户操作。
```

但清单上的每一项，必须由人拿着真实硬件确认。尤其是电源、电压、发热、线材固定这些问题，不能只看文字。

## 15.2 发布前检查分四类

### 1. 硬件安全

- 电压是否匹配。
- GND 是否连接正确。
- 外接电源是否稳定。
- 线材是否松动。
- 执行器是否发热。
- 是否存在短路风险。

### 2. 程序可靠性

- 开机是否能正常启动。
- 传感器读取失败时是否有提示。
- 网络断开时是否能重连或降级。
- 执行器是否有停止条件。
- 按键是否有消抖。

### 3. 用户体验

- 屏幕显示是否清楚。
- 串口日志是否能帮助排查。
- 按键操作是否符合直觉。
- 报警提示是否明显。
- 说明文档是否能让别人复现。

### 4. 项目资料

- 自然语言需求是否保存。
- 连线图是否清楚。
- 程序截图或代码是否保存。
- 测试结果是否记录。
- 已知问题是否说明。

## 15.3 实验项目：给环境监测站做发布前检查

提示词：

```text
我的环境监测站项目包含 Arduino Uno、DHT20、光敏传感器、OLED 和报警 LED。请帮我生成发布前检查清单。

要求：
1. 按硬件安全、程序可靠性、显示和交互、测试记录四类整理。
2. 每一项都要能人工确认。
3. 请加入传感器断开、光照变化、温度阈值、OLED 不显示、串口无输出等异常测试。
4. 请给出一份展示前的最终检查表。
```

## 15.4 故障演练

你可以主动做几个故障演练，并用串口、连线图和 AI 排查：

- 拔掉传感器 GND。
- 改错 OLED 地址。
- 把阈值设置得不合理。
- 把串口波特率选错。
- 断开 WiFi。

你要记录：现象是什么、串口输出是什么、怀疑原因是什么、最终怎么修复。

这比只看项目成功运行更能训练工程能力。

## 15.5 让 AI 生成测试用例

自然语言测试用例可以这样写：

```text
请为我的智能灯控项目生成测试用例。项目有 AUTO 和 MANUAL 两种模式，按键短按切换灯状态，长按切换模式，光敏传感器低于阈值时自动开灯。请用表格列出测试步骤、预期结果和通过标准。
```

测试用例表格示例：

| 测试项 | 操作 | 预期结果 | 是否通过 |
| --- | --- | --- | --- |
| 开机状态 | 上电 | 串口输出启动日志，进入 AUTO |  |
| 光照触发 | 遮住传感器 | AUTO 模式下 LED 点亮 |  |
| 短按按键 | 短按一次 | MANUAL 模式下 LED 状态切换 |  |
| 长按按键 | 长按 2 秒 | 模式切换 |  |
| 传感器异常 | 断开传感器 | 显示错误并关闭执行器 |  |

## 15.6 项目说明文档

一个完整项目至少应该说明：

1. 项目目标。
2. 使用硬件。
3. 接线说明。
4. 程序功能。
5. 如何编译和上传。
6. 如何测试。
7. 常见问题。
8. 安全注意事项。

可以让 AI 先生成草稿：

```text
请根据我的项目需求、连线说明和测试结果，生成一份面向同学的项目说明文档。语言要简单，包含硬件清单、接线表、操作步骤、测试方法和安全提醒。
```

## 15.7 最终展示建议

展示项目时，不建议只展示“成品运行”。更好的展示顺序是：

1. 我想解决什么问题。
2. 我用了哪些硬件。
3. 我怎样用自然语言描述需求。
4. AI 帮我生成了哪些内容。
5. 我人工检查了哪些地方。
6. 我遇到过什么问题，怎样排查。
7. 最终作品演示。

这样能体现你的思考过程，而不是只看结果。

## 15.8 安全红线

入门阶段建议明确这些红线：

- 不直接连接市电负载。
- 不让电机、舵机、灯带直接从 IO 引脚取电。
- 不把未知电压接入开发板引脚。
- 不在无人看管时长时间运行发热项目。
- 不公开真实 WiFi 密码、Token 或个人数据。

## 15.9 本章练习

1. 给环境监测站生成发布前检查清单。
2. 给智能灯控项目生成测试用例表。
3. 故意制造一个错误，记录排查过程。
4. 写一份项目说明文档。
5. 准备一次 3 分钟项目展示。

## 15.10 本章小结

可靠性和安全是硬件项目最后但非常重要的一步。自然语言和 AI 可以帮助你列清单、写测试、整理文档，但真实确认必须由开发者完成。

到这里，你已经走完整个自然语言硬件开发路径：从一句需求开始，到 GPIO、传感器、执行器、通信、显示、联网，再到综合项目和发布检查。未来做更复杂的作品，也可以沿用同样的方法：先说清楚，再生成草稿，再检查验证，再持续迭代。
