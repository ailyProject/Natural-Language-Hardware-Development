# 第 13 章：综合项目一：环境监测站

## 本章要解决什么问题

前面我们分别学了传感器、显示、串口、阈值判断和报警。本章把它们组合起来，完成一个环境监测站。

综合项目的重点不是堆模块，而是学会拆任务：先读数据，再显示数据，再报警，再测试，再优化。

> 图片占位 13-1：环境监测站项目结构图，包含开发板、温湿度传感器、光敏传感器、OLED、LED 或蜂鸣器。

## 13.1 项目目标

本项目实现一个基础环境监测站：

- 每 2 秒读取温度、湿度和光照。
- 在 OLED 上显示当前数据。
- 串口输出完整日志。
- 温度过高或光线过暗时给出提醒。
- 传感器异常时显示错误状态。

这个项目适合作为阶段性作品，也适合在此基础上继续扩展。

## 13.2 硬件建议

可以根据学校器材选择模块。一个常见组合是：

- ESP32 或 Arduino Uno。
- DHT20 温湿度传感器。
- 模拟光敏传感器。
- I2C OLED 显示屏。
- LED 或蜂鸣器作为报警输出。

如果使用 Arduino Uno，I2C 设备通常接 A4/A5；如果使用 ESP32，要根据实际开发板引脚图选择 SDA/SCL。

## 13.3 先把项目拆成任务

不要直接对 AI 说“做一个环境监测站”。先拆成几个小任务：

1. 创建项目并确认开发板。
2. 读取温湿度。
3. 读取光照值。
4. OLED 显示数据。
5. 串口输出日志。
6. 设置报警规则。
7. 处理异常状态。
8. 编写测试清单。

## 13.4 总提示词

```text
我想做一个环境监测站项目，请帮我分步骤生成。

硬件条件：
1. 开发板使用 Arduino Uno。
2. DHT20 温湿度传感器使用 I2C，SDA 接 A4，SCL 接 A5。
3. OLED 屏幕使用 I2C，地址可能是 0x3C。
4. 光敏传感器 AO 接 A0。
5. 报警 LED 使用 D9，并串联 220 欧姆电阻。

程序行为：
1. 每 2 秒读取一次温度、湿度和光照值。
2. OLED 第一行显示温度，第二行显示湿度，第三行显示光照值，第四行显示状态。
3. 串口输出完整数据日志。
4. 温度超过 30 度或光照低于阈值时点亮报警 LED。
5. 如果传感器读取失败，OLED 显示 SENSOR ERROR，串口输出错误信息。

请先给出任务拆分和接线检查清单，再生成 Blockly 程序逻辑和 Arduino/C++ 代码思路。不要一次跳过验证步骤。
```

这个提示词强调“先拆分、再生成”，适合用在综合项目中。

## 13.5 第一步：只读传感器

综合项目不要一开始就加屏幕和报警。先让传感器数据通过串口输出。

提示词：

```text
请先只完成环境监测站的传感器读取部分。每 2 秒读取温度、湿度和光照值，并输出到串口。暂时不要加入 OLED 和报警 LED。
```

这一阶段成功标准：串口能稳定输出数据，数据大致合理。

## 13.6 第二步：加入 OLED 显示

传感器读数稳定后，再加入 OLED。

提示词：

```text
在当前项目基础上加入 OLED 显示。请保留原来的串口输出。OLED 显示温度、湿度、光照值和状态。请不要改变传感器读取逻辑。
```

“保留串口输出”很重要。即使有屏幕，串口仍然是排查问题的好工具。

## 13.7 第三步：加入报警规则

报警规则要明确：

```text
请在当前环境监测站中加入报警规则：温度超过 30 度，或者光照平均值低于 400 时，D9 报警 LED 点亮；否则熄灭。串口输出 alarm=ON 或 alarm=OFF。请把阈值定义成变量，方便后续调整。
```

阈值不一定一次就准确，你需要根据串口数据调整。

## 13.8 测试清单

| 测试项 | 操作 | 预期结果 |
| --- | --- | --- |
| 温湿度读取 | 正常放置传感器 | 串口和 OLED 显示合理数值 |
| 光照变化 | 遮住光敏传感器 | 光照值明显变化 |
| 温度报警 | 修改阈值或模拟高温 | 报警 LED 点亮 |
| 传感器异常 | 暂时断开传感器 | 显示 SENSOR ERROR |
| 串口日志 | 打开串口工具 | 输出温度、湿度、光照和报警状态 |

## 13.9 常见问题

### OLED 和 DHT20 都是 I2C，会不会冲突

只要地址不同，I2C 允许多个设备共用 SDA 和 SCL。可以先运行 I2C 扫描确认地址。

### OLED 显示正常，但传感器无数据

说明 I2C 总线可能通了，但传感器地址、库或初始化有问题。

### 报警一直触发

先看串口数据，再判断阈值是否设置不合理，或者光敏数值方向是否反了。

## 13.10 拓展方向

1. 增加按键切换显示页面。
2. 增加蜂鸣器报警。
3. 把数据上传到 MQTT 平台。
4. 增加数据平均滤波。
5. 做一个外壳或展示板。

## 13.11 本章小结

综合项目不是把所有功能一次性生成出来，而是把项目拆成能验证的小步骤。每一步都通过串口、屏幕或硬件现象确认，再进入下一步。

自然语言在综合项目里最有价值的地方，是帮助你拆任务、生成检查清单、记录测试结果，而不是只生成一大段看不懂的代码。
