# 第 9 章：常见通信接口：I2C、SPI 和 UART

## 本章要解决什么问题

传感器、显示屏、无线模块和开发板之间需要“说话”。不同模块使用不同的通信方式，最常见的是 I2C、SPI 和 UART。

这三个词看起来有点专业，但入门时不用先背协议细节。先把它们理解成三种接线和交流方式：I2C 适合多个简单模块共用两根信号线，SPI 速度快但线更多，UART 常用于一对一串口通信。

## 9.1 I2C：两根线挂多个设备

I2C 常见信号线是：

- SDA：数据线。
- SCL：时钟线。

很多温湿度传感器、OLED 屏幕、扩展芯片都使用 I2C。I2C 设备通常还有一个地址，开发板通过地址区分不同设备。

在 Arduino Uno 上，I2C 常见接法是：

```text
SDA -> A4
SCL -> A5
VCC -> 5V 或 3.3V
GND -> GND
```

不同开发板的 I2C 引脚可能不同，所以要看引脚图。

## 9.2 SPI：速度快，线更多

SPI 常见信号线是：

- MOSI：主设备发给从设备。
- MISO：从设备发给主设备。
- SCK：时钟线。
- CS 或 SS：片选线，用来选择当前通信的设备。

SPI 常见于显示屏、存储卡、无线模块等。它速度快，但接线比 I2C 多。

## 9.3 UART：TX 和 RX 要交叉

UART 常用于串口模块，例如 GPS、蓝牙串口模块、某些通信模块。

UART 最容易记错的是 TX 和 RX：

```text
模块 TX -> 开发板 RX
模块 RX -> 开发板 TX
GND -> GND
```

TX 是发送，RX 是接收。一个设备发送的数据，要接到另一个设备的接收脚上。

## 9.4 用自然语言描述通信模块

描述通信接口时，提示词要说清楚：

1. 模块名称和接口类型。
2. 每根线接到哪个引脚。
3. 电压要求。
4. 是否需要地址、波特率或片选引脚。
5. 期望读取或发送什么数据。

示例：

```text
我使用 Arduino Uno 和一个 I2C OLED 屏幕。OLED 的 SDA 接 A4，SCL 接 A5，VCC 接 5V，GND 接 GND。请先生成 I2C 扫描程序，帮我确认设备地址，然后再生成显示文字的 Blockly 程序。
```

## 9.5 实验项目一：I2C 扫描

I2C 扫描是排查 I2C 模块的好方法。它会扫描总线上有哪些设备地址。

### 提示词

```text
我使用 Arduino Uno，需要检查 I2C 总线上是否能找到设备。请生成一个 I2C 扫描程序。

要求：
1. 串口波特率 9600。
2. 扫描 0x01 到 0x7F 的 I2C 地址。
3. 找到设备时在串口输出地址。
4. 没找到设备时提示检查 SDA、SCL、VCC、GND。
5. 请说明 Arduino Uno 的 SDA 和 SCL 常见引脚。
```

> 图片占位 9-3：I2C 扫描串口输出截图，显示找到地址 0x3C 或其他地址。

### 学习重点

I2C 扫描不是最终项目，但它非常适合用来排查：如果扫描不到设备，不要急着改显示代码，先检查接线、供电和地址。

## 9.6 实验项目二：UART 模块收发

UART 项目要特别注意波特率和 TX/RX 交叉。

提示词示例：

```text
我使用 Arduino Uno 和一个 UART 蓝牙串口模块。模块 TX 接 Arduino 的 RX，模块 RX 接 Arduino 的 TX，GND 共地。模块默认波特率 9600。请生成一个串口透传测试程序，让电脑串口输入的内容发送给模块，同时把模块返回的数据输出到串口。请提醒我上传程序时硬件串口被占用可能导致上传失败。
```

如果你使用 Arduino Uno 的 D0/D1 硬件串口，要注意上传时可能需要拔掉模块。也可以使用软串口方案，但要根据开发板和项目条件决定。

## 9.7 实验项目三：SPI 设备入门

SPI 设备类型很多，不同模块库差异较大。入门时可以先让 AI 根据模块说明生成接线表，而不是立刻写复杂程序。

提示词示例：

```text
我有一个 SPI 显示屏模块，请根据它的引脚名称帮我整理 Arduino Uno 接线表。模块引脚包括 VCC、GND、SCK、MOSI、CS、DC、RST。请说明哪些是 SPI 固定引脚，哪些是可以在程序中指定的控制引脚，并提醒我检查模块电压是 3.3V 还是 5V。
```

## 9.8 通信接口检查清单

- 模块接口类型是否确认。
- VCC 电压是否匹配。
- GND 是否连接。
- I2C 的 SDA/SCL 是否接对。
- I2C 地址是否确认。
- UART 的 TX/RX 是否交叉。
- UART 波特率是否一致。
- SPI 的 MOSI/MISO/SCK/CS 是否接对。
- 是否安装了对应库。
- 串口是否输出了调试信息。

## 9.9 常见问题

### I2C 扫描不到设备

先查 SDA、SCL、VCC、GND，再查地址和模块电压。

### UART 没数据

检查 TX/RX 是否接反，波特率是否一致，GND 是否共地。

### 上传失败

如果模块占用了 D0/D1，可能影响上传。先断开模块再上传。

### SPI 屏幕白屏

检查电压、库型号、CS/DC/RST 引脚和初始化参数。

1. 运行 I2C 扫描程序，记录设备地址。
2. 让 AI 根据一个模块的引脚名称生成接线表。
3. 画出 UART TX/RX 交叉连接图。
4. 对比 I2C 和 SPI 的接线数量。
5. 写一段问题描述，让 AI 排查“I2C 扫描不到设备”。

## 9.11 本章小结

通信接口是开发板和模块之间的语言。入门时不用把协议细节一次弄懂，先掌握每种接口的接线特点、常见参数和排查方法。

自然语言在这里特别适合做“接线表生成”和“错误排查”。但每根线最后还是要你自己确认。
