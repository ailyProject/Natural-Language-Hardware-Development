# 第 10 章：显示屏、矩阵灯和人机反馈

## 本章要解决什么问题

到目前为止，我们主要通过串口看硬件状态。但真实作品通常不能一直连着电脑看串口。显示屏、LED 点阵、RGB 矩阵这些输出设备，可以让硬件自己把状态展示出来。

本章学习如何用自然语言描述显示内容：显示什么、放在哪里、什么时候更新、不同状态怎么显示。

## 10.1 显示也是一种调试

显示屏不只是“好看”。它能告诉用户：

- 当前温度是多少。
- 当前模式是什么。
- 是否连接网络。
- 是否传感器异常。
- 是否正在执行某个动作。

如果项目离开电脑也能显示状态，就更像一个完整作品。

## 10.2 常见显示设备

入门课程里常见显示设备包括：

- OLED 屏：常用 I2C，适合显示文字和简单图形。
- LCD 屏：适合显示固定行列文字。
- LED 点阵：适合显示图案、数字、滚动文字。
- RGB 矩阵或灯带：适合做颜色状态反馈。

不同显示设备需要不同库，接线和初始化方式也不同。让 AI 生成程序前，要先告诉它具体模块型号。

## 10.3 用自然语言描述显示内容

显示类项目要说清楚：

1. 显示设备型号和接口。
2. 屏幕尺寸或分辨率。
3. 每一行或每一区域显示什么。
4. 多久刷新一次。
5. 异常状态怎么显示。
6. 是否需要滚动、图标或颜色。

示例：

```text
我使用 Arduino Uno 和 I2C OLED 屏幕，地址可能是 0x3C，分辨率 128x64。请在第一行显示“Temp: xx C”，第二行显示“Hum: xx %”，第三行显示当前状态。如果传感器读取失败，请显示“SENSOR ERROR”。
```

## 10.4 实验项目一：OLED 显示温湿度

### 提示词

```text
我使用 Arduino Uno、DHT20 温湿度传感器和 I2C OLED 屏幕做一个数据显示项目。

硬件连接：
1. DHT20 和 OLED 都使用 I2C。
2. SDA 接 A4，SCL 接 A5，VCC 和 GND 按模块要求连接。
3. OLED 地址可能是 0x3C。

程序行为：
1. 每 2 秒读取一次温度和湿度。
2. OLED 第一行显示温度。
3. OLED 第二行显示湿度。
4. OLED 第三行显示状态：NORMAL 或 SENSOR ERROR。
5. 串口同步输出温湿度和状态。

请生成 Blockly 程序逻辑，说明需要安装哪些库，解释 Arduino/C++ 代码思路，并给出 I2C 地址排查方法。
```

## 10.5 显示布局先画草图

显示项目最怕“全都想显示”，结果屏幕很乱。你可以先画一个小草图：

```text
第 1 行：温度
第 2 行：湿度
第 3 行：状态
第 4 行：报警提示
```

再把草图交给 AI：

```text
请根据这个 OLED 布局草图生成显示逻辑。屏幕较小，文字要简短，不要在同一行显示太多内容。
```

## 10.6 实验项目二：LED 矩阵显示图案

LED 矩阵适合做表情、箭头、数字、简单动画。

提示词示例：

```text
我使用一个 8x8 LED 点阵模块。请帮我设计三个图案：笑脸、警告、对勾。请用适合 Blockly 的方式生成图案数据，并说明如何切换显示。按键短按切换下一个图案，长按清空显示。
```

矩阵项目能训练你把“想显示的图案”变成具体像素数据，也很适合项目展示。

## 10.7 实验项目三：RGB 灯带做状态反馈

RGB 灯带可以用颜色表示状态：

- 绿色：正常。
- 黄色：警告。
- 红色：异常。
- 蓝色：联网中。

提示词示例：

```text
我使用 Arduino Uno 和一条可编程 RGB 灯带，数据线接 D6，外接 5V 电源供电并与 Arduino 共地。请根据温度状态改变颜色：低于 30 度显示绿色，30 到 35 度显示黄色，高于 35 度显示红色。请特别提醒灯带电源不要直接从开发板 IO 引脚获取。
```

## 10.8 显示项目检查清单

- 显示模块型号是否明确。
- 接口类型是否确认。
- VCC 电压是否匹配。
- I2C 地址是否扫描确认。
- 屏幕尺寸是否正确。
- 刷新频率是否合适。
- 文字是否太长。
- 异常状态是否有显示。
- 串口是否保留调试输出。

## 10.9 常见问题

### OLED 不显示

检查 I2C 接线、地址、库、屏幕供电和初始化参数。

### 屏幕显示乱码或错位

检查分辨率、字体、换行和清屏逻辑。

### LED 矩阵方向反了

有些模块坐标方向不同，可以让 AI 生成翻转或旋转后的图案数据。

### 灯带颜色不对

可能是 RGB 顺序不同，常见有 RGB、GRB 等。让 AI 根据灯带型号调整颜色顺序。

## 10.10 本章练习

1. 把温湿度数据显示到 OLED 上。
2. 设计一个 8x8 图案，让 AI 转成矩阵数据。
3. 用 RGB 灯颜色表示三种状态。
4. 给显示项目增加“SENSOR ERROR”异常页面。
5. 让 AI 根据你的屏幕草图生成显示逻辑。

## 10.11 本章小结

显示设备让硬件作品能离开电脑独立表达状态。自然语言特别适合描述显示布局、状态文字和颜色规则。

但显示不是越多越好。好显示应该简洁、清楚、能帮助用户判断设备现在在做什么。
