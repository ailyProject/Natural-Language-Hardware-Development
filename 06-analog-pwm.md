# 第 6 章：模拟输入和 PWM 输出

## 本章要解决什么问题

前面几章里，我们主要处理“开”和“关”：LED 亮或灭，按键按下或松开。这类信号很好理解，但真实硬件里还有很多连续变化的量，比如亮度、温度、旋钮位置、电机速度、舵机角度。

本章先学习两件事：模拟输入和 PWM 输出。简单说，模拟输入负责“读一个变化的数”，PWM 输出负责“用快速开关模拟出不同强度”。

> 图片占位 6-1：从电位器输入模拟值，再控制 LED 亮度的流程图。

## 6.1 什么是模拟输入

模拟输入常用来读取连续变化的电压。比如电位器中间脚接到 A0，旋钮转动时，A0 读到的数值会变化。

在 Arduino Uno 上，`analogRead(A0)` 常见返回范围是 0 到 1023：

- 接近 0：电压接近 GND。
- 接近 1023：电压接近 5V。
- 中间值：电压在中间。

不同开发板的范围可能不同，比如有的板是 0 到 4095。学习时不要死记一个数，要理解“它代表一个范围”。

## 6.2 什么是 PWM

很多开发板不能直接输出任意模拟电压，但可以用 PWM 快速开关。开得时间长一点，LED 看起来更亮；开得时间短一点，LED 看起来更暗。

在 Arduino Uno 上，`analogWrite()` 常用范围是 0 到 255：

- 0：几乎一直关。
- 255：几乎一直开。
- 128：大约一半时间开，一半时间关。

注意：Arduino Uno 不是所有引脚都支持 PWM，常见 PWM 引脚是 D3、D5、D6、D9、D10、D11。

## 6.3 用自然语言描述连续变化

做模拟量项目时，提示词要说清楚：

1. 读取哪个传感器或输入模块。
2. 输入接在哪个模拟引脚。
3. 输出接在哪个 PWM 引脚。
4. 输入范围如何映射到输出范围。
5. 串口要输出哪些数值。

示例：

```text
我使用 Arduino Uno。电位器中间脚接 A0，两端分别接 5V 和 GND。LED 接 D9，D9 是 PWM 引脚。请读取电位器数值，并把 0 到 1023 映射成 0 到 255，用来控制 LED 亮度。串口输出原始值和 PWM 值。
```

这段需求把硬件、引脚、范围、映射和调试方式都说清楚了。

## 6.4 实验项目一：电位器控制 LED 亮度

### 硬件准备

- Arduino Uno 或兼容开发板。
- 电位器一个。
- LED 一个。
- 220 欧姆或 330 欧姆电阻一个。
- 杜邦线若干。

### 接线说明

```text
电位器一端 -> 5V
电位器另一端 -> GND
电位器中间脚 -> A0
D9 -> 电阻 -> LED 长脚
LED 短脚 -> GND
```

> 图片占位 6-3：电位器接 A0，LED 接 D9 的完整接线图。

### 提示词

```text
我使用 Arduino Uno 做电位器控制 LED 亮度实验。

硬件连接：
1. 电位器两端分别接 5V 和 GND。
2. 电位器中间脚接 A0。
3. 外接 LED 接 D9，D9 先接 220 欧姆电阻，再接 LED 长脚，LED 短脚接 GND。

程序行为：
1. 读取 A0 的模拟值。
2. 把 0 到 1023 映射成 0 到 255。
3. 用映射后的 PWM 值控制 D9 上的 LED 亮度。
4. 串口输出原始模拟值和 PWM 值。

请生成 Blockly 程序逻辑，解释 Arduino/C++ 代码思路，并给出测试步骤。
```

### 代码思路

```cpp
const int sensorPin = A0;
const int ledPin = 9;

void setup() {
  Serial.begin(9600);
  pinMode(ledPin, OUTPUT);
}

void loop() {
  int analogValue = analogRead(sensorPin);
  int pwmValue = map(analogValue, 0, 1023, 0, 255);
  analogWrite(ledPin, pwmValue);

  Serial.print("analog=");
  Serial.print(analogValue);
  Serial.print(", pwm=");
  Serial.println(pwmValue);

  delay(100);
}
```

## 6.5 实验项目二：光线越暗，灯越亮

电位器是人工输入，光敏传感器则能读取环境变化。这个项目更接近真实应用：环境越暗，LED 越亮。

提示词示例：

```text
我使用 Arduino Uno 和一个模拟光敏传感器模块。传感器 AO 接 A0，VCC 接 5V，GND 接 GND。LED 接 D9。请让环境越暗 LED 越亮，环境越亮 LED 越暗。串口输出光敏原始值和 LED PWM 值。如果不同模块数值方向相反，请告诉我如何反转映射。
```

这里特意加了“数值方向相反”。因为有些光敏模块是越亮数值越大，有些电路可能刚好相反。让 AI 说明如何判断和反转，比直接假设更稳。

## 6.6 映射不是魔法

映射就是把一个范围换成另一个范围。例如：

```text
0 到 1023 -> 0 到 255
```

如果希望方向反过来，就写成：

```text
0 到 1023 -> 255 到 0
```

自然语言里可以直接说：

```text
请把传感器值越大映射成 LED 越暗。
```

AI 通常会帮你生成反向映射，但你仍然要通过串口看原始值，确认方向是否符合真实模块。

## 6.7 常见问题

### LED 亮度不变化

检查 LED 是否接在支持 PWM 的引脚上。Arduino Uno 的 D8 不能用 `analogWrite()` 调光，D9 可以。

### 串口值一直不变

检查模拟输入引脚是否接对，传感器或电位器是否有供电，GND 是否连接。

### 亮暗方向反了

把映射范围反过来，或让 AI 根据串口数据帮你修改。

### LED 闪烁明显

可能是输出变化太快、供电不稳或程序里加入了不合适的延时。先通过串口观察原始值是否剧烈波动。

## 6.8 本章练习

1. 用电位器控制蜂鸣器提示频率或响停间隔。
2. 用光敏传感器控制 LED，当光线低于阈值时点亮。
3. 让 AI 把电位器控制 LED 的程序改成“分三档亮度”。
4. 观察串口中的原始值，写出你自己的传感器取值范围。

## 6.9 本章小结

本章从“开关控制”进入了“连续控制”。模拟输入让开发板能读取变化，PWM 输出让开发板能控制亮度、速度和强度。

用自然语言描述这类项目时，最重要的是说清楚输入范围、输出范围和映射关系。真实模块的数值方向可能不同，所以一定要用串口看数据，再决定程序怎么改。
