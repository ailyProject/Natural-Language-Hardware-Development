# 第 11 章：联网硬件和物联网应用

## 本章要解决什么问题

当硬件能联网，它就不只是一个本地小装置了。它可以上传传感器数据，接收远程命令，显示天气，控制家里的灯，或者接入 MQTT 平台。

联网项目比前面的项目多了几个新问题：网络配置、连接失败、数据格式、断网重连和隐私信息保护。

## 11.1 物联网项目的基本流程

一个简单物联网项目通常包含：

1. 连接 WiFi 或其他网络。
2. 读取传感器或接收本地输入。
3. 通过 HTTP、MQTT 或蓝牙发送数据。
4. 接收远程指令。
5. 断网时自动重连或进入离线模式。
6. 用串口或屏幕显示连接状态。

这里最重要的不是“把数据发出去一次”，而是让设备在网络不稳定时也能给出清楚状态。

## 11.2 不要把密码写得到处都是

联网项目会涉及 WiFi 名称、密码、服务器地址、Token 等信息。学习和展示时要注意：

- 不要把真实密码截图公开。
- 不要把 Token 发到公开仓库。
- 展示项目可以使用临时 WiFi 或测试账号。
- 提交作业时可以用占位符代替敏感信息。

自然语言提示词里也可以写：

```text
请用占位符 WIFI_SSID 和 WIFI_PASSWORD，不要写入真实密码。
```

## 11.3 用自然语言描述联网任务

联网项目提示词要说清楚：

1. 使用哪块支持联网的开发板。
2. 连接方式，如 WiFi、蓝牙或以太网。
3. 要发送什么数据。
4. 发到哪里，用 HTTP 还是 MQTT。
5. 多久发送一次。
6. 失败后怎么处理。
7. 如何显示连接状态。

示例：

```text
我使用 ESP32 做温湿度上传项目。请连接 WiFi，读取 DHT20 温湿度，每 10 秒通过 MQTT 发布到主题 room/env。连接成功时串口输出 WIFI CONNECTED，MQTT 断开时自动重连。如果读取失败，不发布数据，只输出错误日志。
```

## 11.4 实验项目一：联网状态显示

先不要急着上传数据，第一步先让开发板连接 WiFi，并能显示连接状态。

提示词示例：

```text
我使用 ESP32 做 WiFi 连接测试。

要求：
1. 使用占位符 WIFI_SSID 和 WIFI_PASSWORD。
2. 开机后连接 WiFi。
3. 连接中串口输出“CONNECTING”。
4. 连接成功后输出 IP 地址。
5. 如果 15 秒内连接失败，输出“WIFI FAILED”，并每 5 秒重试一次。

请生成 Blockly 程序逻辑和 Arduino/C++ 代码思路，并说明测试步骤。
```

## 11.5 实验项目二：HTTP 上传数据

HTTP 适合入门理解“向服务器发送一次请求”。可以先用测试接口或练习用服务器。

提示词示例：

```text
我使用 ESP32 和一个温湿度传感器。请每 10 秒读取一次温湿度，并通过 HTTP POST 发送到测试服务器地址 SERVER_URL。请求 JSON 内容包含 temperature、humidity、deviceId 和 timestamp。发送成功输出 HTTP OK，失败输出错误码。请使用占位符，不要写真实服务器地址。
```

这里要重点理解：数据不只是数字，还要有设备编号、时间和发送结果。

## 11.6 实验项目三：MQTT 控制 LED

MQTT 很适合物联网控制。设备订阅一个主题，收到命令后控制输出。

提示词示例：

```text
我使用 ESP32 做 MQTT 控制 LED 项目。设备连接 WiFi 后连接 MQTT 服务器，订阅主题 device/led。收到 ON 时点亮板载 LED，收到 OFF 时熄灭。每次状态变化后发布当前状态到 device/led/status。请加入断线重连，并在串口输出 WiFi 和 MQTT 状态。
```

## 11.7 断网重连和离线状态

真实网络不稳定，所以联网项目要考虑失败情况：

- WiFi 连接失败怎么办？
- MQTT 断开怎么办？
- 服务器没有响应怎么办？
- 传感器读取失败时是否还上传？
- 离线时本地功能是否继续运行？

可以让 AI 加入重连逻辑：

```text
请给当前 MQTT 项目增加断线重连逻辑。WiFi 断开时每 5 秒尝试重连，MQTT 断开时重新连接并重新订阅主题。重连期间 LED 慢闪表示离线状态。
```

## 11.8 联网项目检查清单

- 开发板是否支持联网。
- WiFi 名称和密码是否正确，是否避免公开真实信息。
- 是否有连接状态日志。
- HTTP 或 MQTT 地址是否正确。
- 数据格式是否清楚。
- 是否处理发送失败。
- 是否有断线重连。
- 是否保留本地调试方式。
- 是否避免把 Token 写进公开材料。

## 11.9 常见问题

### WiFi 连不上

检查 SSID、密码、信号强度、开发板是否支持当前频段。很多 ESP32 只支持 2.4GHz WiFi。

### HTTP 请求失败

检查服务器地址、网络连接、请求方法、JSON 格式和证书要求。

### MQTT 连上又断开

检查服务器地址、端口、用户名密码、Client ID 是否重复、心跳设置。

### 上传了数据但平台看不到

检查主题、设备 ID、数据格式和平台规则。

## 11.10 本章练习

1. 让 ESP32 连接 WiFi 并输出 IP 地址。
2. 让 AI 生成 HTTP 上传 JSON 数据的程序。
3. 用 MQTT 控制 LED 开关。
4. 给 MQTT 项目增加断线重连状态提示。
5. 把真实密码替换成占位符，整理一份可公开的项目说明。

## 11.11 本章小结

联网让硬件作品能和外部系统连接，但也带来了网络失败、数据格式和隐私保护问题。

自然语言可以帮助你描述联网流程和异常处理。写提示词时，不要只说“上传数据”，要说清楚上传什么、发到哪里、多久一次、失败后怎么办。
